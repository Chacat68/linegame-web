# 星际贸易商 - 美术与视觉设计

## 系统概述

美术与视觉设计是《星际贸易商》的重要组成部分，负责创造游戏的视觉风格、用户界面设计和整体美学体验。本设计文档详细规划了游戏的艺术方向、视觉风格、UI设计规范和技术实现方案，旨在创造一个既美观又实用的星际贸易世界。

## 艺术方向

### 整体风格定位

1. **科幻现实主义**
   - 基于现实科学的未来科技想象
   - 避免过于奇幻或不切实际的设计
   - 注重功能性和实用性的体现
   - 保持技术感和未来感的平衡

2. **简约现代风格**
   - 清晰简洁的设计语言
   - 避免过度装饰和复杂细节
   - 注重信息传达的效率
   - 符合现代UI设计趋势

3. **多元文化融合**
   - 体现不同星球的文化特色
   - 融合东西方设计元素
   - 展现星际文明的多样性
   - 保持整体风格的统一性

### 色彩方案

1. **主色调**
   - 深蓝色（#1a237e）：代表深邃的太空
   - 银灰色（#90a4ae）：代表科技和金属质感
   - 青色（#00bcd4）：代表能量和活力
   - 橙色（#ff9800）：代表警告和重要信息

2. **辅助色彩**
   - 深紫色（#4a148c）：神秘和高级功能
   - 绿色（#4caf50）：成功和正面反馈
   - 红色（#f44336）：危险和错误提示
   - 黄色（#ffeb3b）：注意和提醒

3. **中性色彩**
   - 深灰色（#263238）：背景和容器
   - 浅灰色（#eceff1）：文本和边框
   - 白色（#ffffff）：高亮和强调
   - 黑色（#000000）：阴影和深度

4. **星球特色色彩**
   - 沙漠星球：暖色调（橙、黄、棕）
   - 海洋星球：蓝色调（蓝、青、绿）
   - 森林星球：绿色调（绿、棕、黄）
   - 冰雪星球：冷色调（白、蓝、灰）
   - 火山星球：红色调（红、橙、黑）

### 视觉层次

1. **信息优先级**
   - 关键信息：高对比度、大字体、醒目颜色
   - 重要信息：中等对比度、标准字体、辅助颜色
   - 次要信息：低对比度、小字体、中性颜色
   - 背景信息：最低对比度、微妙提示

2. **视觉引导**
   - 使用颜色引导用户注意力
   - 通过大小和位置建立层次
   - 利用动画和效果增强交互
   - 保持视觉流的连贯性

## 2D图形设计

### 图标设计规范

1. **设计原则**
   - 简洁明了：一目了然的功能表达
   - 风格统一：保持一致的设计语言
   - 可识别性：在小尺寸下仍清晰可辨
   - 可扩展性：支持不同分辨率和尺寸

2. **图标分类**
   - 功能图标：按钮、菜单、工具等
   - 状态图标：进度、警告、成功等
   - 物品图标：商品、设备、资源等
   - 导航图标：方向、位置、路径等

3. **技术规范**
   - 基础尺寸：32x32像素
   - 扩展尺寸：16x16, 64x64, 128x128像素
   - 格式：SVG矢量格式为主，PNG位图为辅
   - 颜色：支持单色、双色和全彩版本

### 界面元素设计

1. **按钮设计**
   - 主要按钮：高对比度、醒目颜色、明显边框
   - 次要按钮：中等对比度、辅助颜色、细边框
   - 文本按钮：无背景、下划线或颜色区分
   - 图标按钮：图标+文字或纯图标

2. **输入控件**
   - 文本框：清晰边框、占位符文本、焦点状态
   - 下拉菜单：展开指示器、选项高亮、搜索功能
   - 滑块：进度指示、数值显示、拖拽反馈
   - 开关：明确的开/关状态、平滑动画

3. **信息展示**
   - 卡片：圆角、阴影、清晰分组
   - 列表：交替背景、分隔线、排序指示
   - 表格：表头固定、行高亮、数据对齐
   - 图表：清晰标签、颜色编码、交互提示

### 插画和概念图

1. **星球插画**
   - 不同类型星球的特色表现
   - 大气层、地表特征、轨道设施
   - 日夜变化和天气效果
   - 文明程度和发展水平体现

2. **飞船设计**
   - 功能性优先的外观设计
   - 不同等级和类型的差异化
   - 模块化组件的视觉表现
   - 损坏和升级状态的变化

3. **角色设计**
   - 多样化的外星种族
   - 职业和身份的视觉区分
   - 服装和装备的文化特色
   - 表情和动作的情感表达

## UI设计规范

### 布局系统

1. **网格系统**
   - 基础网格：8像素为基本单位
   - 列网格：12列响应式布局
   - 间距规范：8px, 16px, 24px, 32px
   - 对齐方式：左对齐、居中、右对齐

2. **容器设计**
   - 主容器：全屏或固定宽度
   - 内容容器：内边距和最大宽度限制
   - 卡片容器：圆角、阴影、边框
   - 模态容器：遮罩、居中、动画效果

3. **响应式设计**
   - 断点设置：手机(<768px)、平板(768-1024px)、桌面(>1024px)
   - 弹性布局：Flexbox和Grid布局
   - 自适应图片：不同分辨率的图片资源
   - 触摸优化：按钮大小、间距调整

### 字体系统

1. **字体选择**
   - 主字体：Noto Sans（支持多语言）
   - 代码字体：Fira Code（等宽字体）
   - 装饰字体：Orbitron（科幻风格标题）
   - 备用字体：系统默认无衬线字体

2. **字体层级**
   - H1标题：32px, 粗体, 行高1.2
   - H2标题：24px, 粗体, 行高1.3
   - H3标题：20px, 粗体, 行高1.4
   - 正文：16px, 常规, 行高1.5
   - 小字：14px, 常规, 行高1.4
   - 说明：12px, 常规, 行高1.3

3. **文本样式**
   - 强调：粗体或颜色区分
   - 链接：下划线和悬停效果
   - 代码：等宽字体和背景色
   - 引用：斜体和缩进

### 组件库设计

1. **基础组件**
   - Button：各种样式和状态的按钮
   - Input：文本输入框和验证状态
   - Select：下拉选择器和多选
   - Checkbox：复选框和单选框
   - Slider：滑块和范围选择器

2. **复合组件**
   - Card：信息卡片和操作卡片
   - Modal：模态对话框和确认框
   - Tooltip：工具提示和帮助信息
   - Dropdown：下拉菜单和操作菜单
   - Pagination：分页器和页码导航

3. **数据组件**
   - Table：数据表格和排序功能
   - List：列表项和虚拟滚动
   - Tree：树形结构和展开折叠
   - Chart：图表组件和数据可视化
   - Progress：进度条和加载指示器

## 动画和效果

### 动画原则

1. **功能性动画**
   - 提供视觉反馈和状态变化
   - 引导用户注意力和操作流程
   - 增强界面的响应感和活力
   - 避免纯装饰性的无意义动画

2. **性能考虑**
   - 优先使用CSS动画和GPU加速
   - 避免复杂的JavaScript动画
   - 控制同时播放的动画数量
   - 提供动画开关和性能模式

3. **用户体验**
   - 动画时长适中（200-500ms）
   - 缓动函数自然流畅
   - 可中断和可跳过
   - 考虑动画敏感用户的需求

### 交互动画

1. **按钮动画**
   - 悬停：颜色变化、阴影增强
   - 点击：按下效果、涟漪扩散
   - 加载：旋转图标、进度指示
   - 禁用：灰化效果、透明度降低

2. **页面转换**
   - 淡入淡出：平滑的透明度变化
   - 滑动：左右或上下滑动效果
   - 缩放：放大缩小的过渡效果
   - 翻转：3D翻转和旋转效果

3. **数据变化**
   - 数字动画：数值的平滑变化
   - 图表动画：数据的动态更新
   - 列表动画：项目的添加删除
   - 状态动画：状态图标的变化

### 视觉效果

1. **粒子效果**
   - 星空背景：缓慢移动的星点
   - 能量效果：流动的光线和粒子
   - 爆炸效果：碎片和光芒扩散
   - 传送效果：粒子聚集和消散

2. **光影效果**
   - 发光效果：重要元素的光晕
   - 阴影效果：深度和层次感
   - 反射效果：金属和玻璃质感
   - 透明效果：玻璃和全息投影

3. **后处理效果**
   - 模糊效果：背景虚化和焦点
   - 色彩调整：饱和度和对比度
   - 噪点效果：复古和科技感
   - 扫描线：CRT显示器效果

## 技术实现

### Web 渲染与资源系统

1. **2D渲染**
   - 场景渲染：Canvas 2D（简单/高兼容）或 WebGL（更高性能/特效）
   - UI 层：DOM（可访问性更好）或 Canvas UI（风格统一）
   - 图片显示：`<img>` / Canvas drawImage / 纹理贴图
   - 可拉伸边框：CSS 9-slice（`border-image`）或自绘九宫格

2. **主题系统**
   - Design Tokens：颜色/间距/字体/层级统一管理
   - CSS Variables：运行时切换主题（如深色模式）
   - 字体：WOFF/WOFF2 + 子集化（subset）
   - 组件样式：复用的 UI 组件与状态样式

3. **动画系统**
   - UI 动画：CSS Animations/Transitions
   - 游戏动画：Canvas 帧循环（requestAnimationFrame）或基于时间轴的 Tween
   - 复杂动画：独立动画状态机（输入→状态→输出）
   - Shader 特效：WebGL Shader（如需要）

### 资源管理

1. **图片资源**
   - 格式选择：PNG（透明）、JPG（照片）、SVG（矢量）
   - 压缩优化：质量和文件大小平衡
   - 分辨率适配：@1x, @2x, @3x资源
   - 图集打包：减少绘制调用

2. **字体资源**
   - 字体格式：TTF、OTF、WOFF
   - 字符集：支持多语言字符
   - 字体大小：预渲染常用尺寸
   - 字体回退：缺失字符的处理

3. **样式资源**
   - CSS样式：可重用的样式定义
   - 主题变量：颜色和尺寸常量
   - 组件样式：标准化的组件外观
   - 响应式样式：不同屏幕的适配

### 性能优化

1. **渲染优化**
   - 批处理：减少绘制调用
   - 遮挡剔除：不可见元素跳过
   - LOD系统：距离相关的细节级别
   - 纹理压缩：GPU友好的格式

2. **内存优化**
   - 资源池化：重复使用对象
   - 延迟加载：按需加载资源
   - 内存监控：实时使用情况
   - 垃圾回收：及时释放资源

3. **加载优化**
   - 预加载：关键资源提前加载
   - 流式加载：大文件分块加载
   - 缓存策略：智能缓存管理
   - 压缩传输：减少网络开销
