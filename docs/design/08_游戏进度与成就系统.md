# 星际贸易商 - 游戏进度与成就系统

## 系统概述

游戏进度与成就系统是《星际贸易商》的核心激励机制，旨在为玩家提供明确的目标、成长路径和成就感。本设计文档详细规划了游戏的进度追踪、成就解锁和奖励机制，以增强玩家的长期参与度和满足感。

## 游戏进度系统

### 玩家等级

1. **等级结构**
   - 初始等级：见习贸易商（1级）
   - 最高等级：星际贸易巨头（50级）
   - 每级所需经验值呈非线性增长
   - 等级划分为5个主要阶段，每个阶段10级

2. **等级阶段**
   - 阶段1（1-10级）：本地贸易商
     * 专注于单一星系内的贸易活动
     * 学习基本游戏机制和贸易技巧
   - 阶段2（11-20级）：区域贸易商
     * 扩展到邻近星系的贸易
     * 开始建立商业网络和声望
   - 阶段3（21-30级）：星际贸易商
     * 跨越多个星区的贸易活动
     * 拥有多艘飞船和专业船员
   - 阶段4（31-40级）：贸易联盟主
     * 影响星际经济和政治
     * 建立自己的贸易站和基地
   - 阶段5（41-50级）：星际贸易巨头
     * 掌控整个星区的贸易网络
     * 参与塑造星际文明的未来

3. **经验获取**
   - 完成交易：基于利润率和交易难度
   - 探索新星球：基于星球稀有度和危险程度
   - 完成任务：基于任务复杂度和重要性
   - 解锁成就：特定成就提供额外经验值
   - 克服危机：成功应对随机事件和挑战

4. **等级奖励**
   - 新能力解锁：特殊贸易选项、谈判技巧等
   - 资源奖励：信用点、稀有物品、飞船部件等
   - 声望提升：与特定派系的关系改善
   - 新区域访问：高级贸易中心、限制区域等
   - 称号和外观：独特的玩家标识和定制选项

### 技能发展

1. **技能树结构**
   - 贸易技能：价格谈判、市场分析、批量交易等
   - 飞行技能：导航效率、燃料节约、规避危险等
   - 社交技能：声望获取、情报收集、派系关系等
   - 管理技能：船员效率、多船协调、资源优化等
   - 探索技能：扫描效率、资源识别、异常探测等

2. **技能点获取**
   - 每升一级获得1个主要技能点
   - 完成特定任务和挑战获得额外技能点
   - 阅读特殊数据和文献获得专业知识点
   - 反复实践特定活动获得熟练度提升

3. **技能效果**
   - 被动效果：持续性能力提升和效率改进
   - 主动能力：可触发的特殊行动和决策选项
   - 解锁选项：新的游戏内容和互动可能性
   - 组合效果：多个技能协同产生的特殊效果

4. **专精路径**
   - 贸易专家：最大化利润和市场操控
   - 探险家：发现和利用稀有资源和机会
   - 舰队指挥：管理多艘飞船和大规模运输
   - 外交官：利用关系网络和政治影响力
   - 技术专家：优化飞船性能和特殊设备

### 声望系统

1. **派系声望**
   - 主要派系：商业联盟、科学院、安全部队、自由联合等
   - 声望等级：敌对、警惕、中立、友好、盟友、尊崇
   - 声望获取：完成派系任务、有利交易、政治支持等
   - 声望损失：违反规定、支持敌对派系、失败任务等

2. **声望效果**
   - 交易优惠：更好的价格和稀有商品访问
   - 特殊任务：高级和高回报的派系任务
   - 保护和支援：危险区域的派系援助
   - 设施访问：派系专属空间站和服务
   - 技术获取：派系专有技术和设备

3. **声望平衡**
   - 派系冲突：提高某派系声望可能降低敌对派系声望
   - 中立选项：维持多方平衡的外交路线
   - 声望恢复：修复受损关系的任务和行动
   - 隐藏身份：在特定区域临时隐藏派系关联

### 故事进度

1. **主线任务**
   - 分为5-7个主要章节
   - 每个章节有明确的目标和叙事弧
   - 章节完成解锁新区域和游戏机制
   - 多结局设计基于关键决策点

2. **支线故事**
   - 派系特定的故事线
   - 角色关系发展的个人故事
   - 区域特定的历史和事件
   - 玩家选择影响的分支叙事

3. **进度追踪**
   - 任务日志：当前和历史任务记录
   - 故事回顾：已完成叙事的摘要和重点
   - 决策记录：关键选择及其影响
   - 时间线：游戏世界和玩家行动的时间记录

### 核心功能

1. **经验值计算**
    ```ts
    export function calculateExperience(actionType: "trade" | "exploration" | "mission", value: number): number {
       let baseXp = 0;
       switch (actionType) {
          case "trade":
             baseXp = value * 0.1;
             break;
          case "exploration":
             baseXp = value * 5;
             break;
          case "mission":
             baseXp = value * 2;
             break;
       }
       const finalXp = baseXp * getXpMultiplier();
       addExperience(finalXp);
       return finalXp;
    }
    ```

2. **成就检查**
    ```ts
    export function checkAchievements(achievements: Achievements, trackers: Record<string, number>): void {
       for (const key of Object.keys(achievements)) {
          const a = achievements[key];
          if (a.completed) continue;
          const trackerValue = trackers[a.tracker] ?? 0;
          a.progress = Math.min(trackerValue, a.target);
          if (a.progress >= a.target) completeAchievement(a);
       }
    }
    ```

3. **成就完成**
   ```ts
   function completeAchievement(a: Achievement): void {
    if (a.completed) return;
    a.completed = true;
    a.completionDate = new Date().toISOString();
    for (const reward of a.rewards) grantReward(reward);
    onAchievementCompleted?.(a);
    if (a.group) checkAchievementGroup(a.group);
   }

   let onAchievementCompleted: ((a: Achievement) => void) | undefined;
   ```

4. **等级提升**
    ```ts
    export function addExperience(progress: PlayerProgress, amount: number): void {
       progress.experience += amount;
       while (progress.experience >= progress.experienceNext) {
          levelUp(progress);
       }
       onExperienceChanged?.(progress.experience, progress.experienceNext);
    }

    function levelUp(progress: PlayerProgress): void {
       progress.experience -= progress.experienceNext;
       progress.level += 1;
       progress.experienceNext = calculateNextLevelXp(progress.level);
       progress.skillPoints += 1;
       grantLevelRewards(progress.level);
       onLevelUp?.(progress.level);
    }

    let onExperienceChanged: ((exp: number, next: number) => void) | undefined;
    let onLevelUp: ((level: number) => void) | undefined;
    ```

## 平衡与调整

### 进度曲线

1. **经验值平衡**
   - 早期快速进展以建立动力
   - 中期稳定增长保持参与度
   - 后期适度放缓以延长游戏寿命
   - 多样化经验来源避免重复性

2. **技能点分配**
   - 总技能点数少于总技能数，鼓励专精
   - 关键技能早期可获取，高级技能需要投入
   - 技能组合提供协同效应和策略深度
   - 重置选项允许实验和调整

3. **声望获取**
   - 声望增长速度随关系等级降低
   - 派系冲突创造有意义的选择
   - 声望损失比获取更快，增加决策权重
   - 修复关系的机会但需要努力

### 成就难度

1. **分布策略**
   - 约40%易于获得的成就（铜级）
   - 约30%中等难度的成就（银级）
   - 约20%高难度的成就（金级）
   - 约8%专家级成就（白金级）
   - 约2%极限挑战成就（传奇级）

2. **时间投入**
   - 短期成就：单次游戏会话可完成
   - 中期成就：需要数天到数周的游戏时间
   - 长期成就：需要数月的持续游戏
   - 终极成就：需要完整掌握游戏系统

3. **难度调整**
   - 基于玩家数据的目标值调整
   - 考虑不同游戏风格和策略的可行性
   - 确保挑战性但不令人沮丧
   - 定期添加新成就保持新鲜感

### 奖励价值

1. **奖励平衡**
   - 奖励价值与成就难度成正比
   - 避免过早提供游戏破坏性奖励
   - 定制奖励与实用奖励的平衡
   - 独特性与实用性的权衡

2. **长期激励**
   - 阶段性奖励维持长期参与
   - 收集完整性的额外奖励
   - 里程碑庆祝和特殊纪念
   - 社区认可和展示机会
