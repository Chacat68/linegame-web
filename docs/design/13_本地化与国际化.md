# 星际贸易商 - 本地化与国际化

## 系统概述

本地化与国际化系统确保《星际贸易商》能够为全球不同地区的玩家提供符合当地语言、文化和习惯的游戏体验。本设计文档详细规划了多语言支持、文化适配、技术实现和运营策略，旨在打造真正的全球化游戏产品。

## 国际化策略

### 目标市场分析

1. **主要市场**
   - 北美市场：英语为主，注重个人成就
   - 欧洲市场：多语言环境，重视合作
   - 亚洲市场：中文、日文、韩文，社交导向
   - 新兴市场：西班牙语、葡萄牙语、俄语等

2. **文化特征**
   - 西方文化：个人主义、竞争导向

   - 拉丁文化：家庭观念、社交活跃
   - 其他文化：宗教信仰、传统习俗

3. **游戏偏好**
   - 欧美玩家：策略深度、自由度高

## 技术实现

### Web 国际化方案

1. **基础 i18n（示例）**
```ts
// 设置语言
i18n.setLocale("zh-CN");

// 获取翻译文本
const text = i18n.t("MAIN_MENU");

// 带参数的翻译
const formattedText = i18n.t("PLAYER_LEVEL", { level: 10 });
```

2. **翻译资源与管理器（示例）**
```ts
export type Locale = "en" | "zh-CN" | "ja" | "ko";

type Dict = Record<string, string>;

export class I18n {
   private locale: Locale = "en";
   private dicts = new Map<Locale, Dict>();
   private listeners = new Set<() => void>();

   async load(locale: Locale, url: string): Promise<void> {
      const res = await fetch(url);
      const dict = (await res.json()) as Dict;
      this.dicts.set(locale, dict);
   }

   setLocale(locale: Locale): void {
      this.locale = locale;
      for (const cb of this.listeners) cb();
   }

   onChange(cb: () => void): () => void {
      this.listeners.add(cb);
      return () => this.listeners.delete(cb);
   }

   t(key: string, params: Record<string, string | number> = {}): string {
      const dict = this.dicts.get(this.locale) ?? {};
      const template = dict[key] ?? key;
      return template.replace(/\{(\w+)\}/g, (_, name: string) => String(params[name] ?? `{${name}}`));
   }
}

export const i18n = new I18n();
```

3. **动态文本更新（示例）**
```ts
// UI 层在 i18n.onChange 时触发一次全量刷新；
// 或者只刷新当前可见视图/组件。
const unsubscribe = i18n.onChange(() => {
   renderCurrentView();
});
```

### 文本管理工具（建议）

1. **翻译编辑器**
- 作为 Web 项目，更常见做法是：翻译文件使用 JSON（或 CSV）维护，编辑器用独立工具（在线表格/本地脚本/后台管理页面）实现。
- MVP 阶段可先人工维护 JSON 文件，配合简单的校验脚本检查缺失 key。

2. **批量导入导出**
- 推荐在构建/CI 阶段使用 Node 脚本完成 JSON↔CSV 转换，避免把编辑工具与游戏运行时代码耦合。
- 浏览器运行时只负责加载最终产物（如 `public/locales/zh-CN.json`）。
