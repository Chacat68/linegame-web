# 星际贸易商 - 技术架构设计

## 系统概述

本文档详细描述了《星际贸易商》的技术架构设计，包括代码结构、数据管理、性能优化和扩展性考虑。基于 Web（浏览器）运行时的特性，我们采用模块化、可扩展的架构设计，确保游戏的稳定性和可维护性。

## 整体架构

### 架构模式

1. **MVC架构模式**
   - Model：数据模型和业务逻辑
   - View：用户界面和显示层
   - Controller：用户输入处理和逻辑控制

2. **组件化设计**
   - 每个游戏系统作为独立组件
   - 组件间通过事件系统通信
   - 支持热插拔和动态加载

3. **分层架构**
   - 表现层：UI和渲染
   - 业务层：游戏逻辑和规则
   - 数据层：数据存储和管理
   - 基础层：引擎接口和工具

### 核心模块

1. **游戏管理器（GameManager）**
   - 游戏状态管理
   - 场景切换控制
   - 全局事件调度
   - 系统初始化和清理

2. **数据管理器（DataManager）**
   - 游戏数据的加载和保存
   - 配置文件管理
   - 缓存机制
   - 数据验证和备份

3. **事件系统（EventSystem）**
   - 全局事件总线
   - 事件订阅和发布
   - 异步事件处理
   - 事件优先级管理

4. **资源管理器（ResourceManager）**
   - 资源加载和卸载
   - 内存池管理
   - 资源预加载
   - 动态资源更新

## 数据架构

### 数据存储策略

1. **配置数据**
   - 使用JSON格式存储静态配置
   - 支持热重载和动态更新
   - 版本控制和兼容性检查

2. **游戏存档**
   - 使用 JSON 数据模型（带 schema/version）
   - 二进制格式提高性能
   - 增量保存减少IO开销
   - 多槽位存档支持

3. **缓存机制**
   - 内存缓存热点数据
   - LRU算法管理缓存
   - 异步加载非关键数据
   - 预测性缓存

### 数据模型设计

1. **基础数据类**
```ts
export interface GameData {
   id: string;
   version: number;
   timestamp: number;
}

export function validateGameData(data: GameData): boolean {
   return Boolean(data.id) && Number.isFinite(data.version) && Number.isFinite(data.timestamp);
}
```

2. **数据访问层**
```ts
export class DataAccessLayer {
   private static instance: DataAccessLayer | null = null;
   private cache = new Map<string, GameData>();
   private dirtyFlags = new Set<string>();

   static getInstance(): DataAccessLayer {
      if (!DataAccessLayer.instance) DataAccessLayer.instance = new DataAccessLayer();
      return DataAccessLayer.instance;
   }

   getData(id: string): GameData | undefined {
      return this.cache.get(id);
   }

   saveData(data: GameData): void {
      this.cache.set(data.id, data);
      this.dirtyFlags.add(data.id);
   }
}
```

## 技术选型说明

> **实现说明**：当前仓库采用 **Vanilla JavaScript + ES Modules** 实现（无构建工具，无 TypeScript 编译），不是下文描述的 TypeScript 技术栈。本节内容为原始设计意图，与当前实现存在偏差，详情参见 [实现方案.md](./实现方案.md) 与 [设计实现对照表.md](./设计实现对照表.md)。

### Web（浏览器）技术优势

1. **开发与迭代效率**
   - 前端生态成熟，开发/调试工具链完善
   - 热更新/快速迭代便于原型验证
   - UI 与数据可视化实现成本较低

2. **性能特点**
   - Canvas/WebGL 在 2D 场景下可满足多数需求
   - 浏览器具备较好的 JIT 与图形加速能力
   - 可按需加载资源，降低首屏与内存压力

3. **跨平台支持**
   - 天然跨平台：Windows/macOS/Linux + 多数移动设备
   - 分发成本低：URL 即用，便于灰度与回滚
   - 可结合 PWA 支持离线与安装体验（可选）

### 技术栈选择（设计目标）

> **注意**：当前实现使用 Vanilla JavaScript + ES Modules，而非下述 TypeScript 方案。

1. **编程语言**
   - TypeScript：主要开发语言（设计目标；当前实现为 Vanilla JS）
   - JavaScript：运行时与第三方库生态

2. **数据格式**
   - JSON：配置文件（静态数据以 `.js` 模块形式定义，如 `js/data/*.js`）
   - localStorage：本地持久化（当前实现；IndexedDB 为后续迁移目标）
   - 二进制（ArrayBuffer/DataView）：高性能序列化（如关卡，暂未实现）

3. **工具链**
   - Git：版本控制
   - CI/CD：自动化流程
   - 性能分析工具：浏览器 Performance 面板、Lighthouse
