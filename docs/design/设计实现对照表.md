## 六、尚未实现的关键系统（按 MVP 优先级）

### P0 — MVP 必需（否则无法形成闭环）

| 系统 | 设计来源 | 建议模块（Web） | 影响 |
|------|----------|------------------|------|
| **核心数据模型（PlayerData / GameState）** | 贯穿所有设计 | `src/models/`（如 `PlayerData.ts`） | 没有统一数据结构会导致 UI/系统/存档无法对齐 |
| **存档读写完整链路** | `存档系统设计.md` | `src/save/SaveManager.ts` | 游戏进度无法持久化（IndexedDB） |
| **基础贸易闭环** | `02_贸易系统.md` | `src/systems/trade/` | 无法验证“买卖获利”的核心玩法 |

### P1 — 重要增强（提升可玩性与留存）

| 系统 | 设计来源 | 建议模块（Web） | 影响 |
|------|----------|------------------|------|
| **动态价格算法** | `05_经济与市场系统.md` | `src/systems/market/PriceCalculator.ts` | 市场静态，交易策略深度不足 |
| **游戏进程 / 声望等级** | `17_游戏进程设计.md` | `src/systems/progression/` | 玩家成长反馈弱 |
| **新手引导系统** | `新手引导设计.md` | `src/systems/tutorial/` | 新玩家理解成本高 |
| **星球经济类型与商品供需绑定** | `04_星球与探索系统.md` | `src/systems/planet/` + `src/systems/market/` | 路线规划缺少逻辑支撑 |

## 八、建议下一步（面向 Web 实现）

1. **立即**：确定数据模型最小集合（`PlayerData`/`GameState`/`SaveMeta`），并在文档中冻结 `schemaVersion`。
2. **本周**：打通存档链路（IndexedDB slot 存档 + JSON 导出/导入可选），确保数据可持久化。
3. **下周**：实现贸易系统最小闭环（市场列表→买入→航行→卖出→结算），并接入基础 UI。
4. **随后**：补齐动态价格算法 + 进程/声望 + 新手引导，形成可玩性与留存。
