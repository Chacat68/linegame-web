# 星际贸易商 - 音效与音乐设计

## 系统概述

音效与音乐系统是《星际贸易商》沉浸式体验的重要组成部分，通过精心设计的音频内容来增强游戏的氛围感、反馈感和情感共鸣。本设计文档详细规划了游戏的音乐风格、音效设计、动态音频系统和技术实现方案。

## 音乐设计

### 音乐风格定位

1. **主要风格**
   - 科幻电子音乐为主基调
   - 融合管弦乐元素增加史诗感
   - 环境音乐营造太空氛围
   - 适度的合成器和数字音效

2. **情感基调**
   - 探索感：神秘、好奇、发现的喜悦
   - 商业感：繁忙、效率、成功的满足
   - 冒险感：紧张、刺激、未知的挑战
   - 宏大感：浩瀚宇宙、星际文明的壮丽

3. **文化融合**
   - 不同星球的音乐特色
   - 异域风情的乐器和旋律
   - 未来主义与传统音乐的结合
   - 多元文化的和谐统一

### 音乐分类系统

1. **主题音乐**
   - 主菜单音乐：宏大、欢迎、期待
   - 游戏主题曲：代表游戏精神的核心旋律
   - 结局音乐：成就感、满足感、回味
   - 特殊事件音乐：重要时刻的音乐强调

2. **环境音乐**
   - 太空航行：宁静、深邃、神秘
   - 星球表面：根据星球类型变化
   - 贸易站：繁忙、商业、人声嘈杂
   - 探索区域：未知、紧张、发现

3. **功能音乐**
   - 界面音乐：轻松、专注、不干扰
   - 战斗音乐：紧张、激烈、节奏感强
   - 成功音乐：胜利、成就、满足
   - 失败音乐：挫折、反思、重新开始

4. **动态音乐**
   - 分层音乐：根据游戏状态添加/移除音轨
   - 过渡音乐：场景切换时的平滑过渡
   - 交互音乐：响应玩家行为的音乐变化
   - 适应性音乐：根据游戏进度调整复杂度

### 具体音乐设计

1. **主菜单音乐**
   - 时长：2-3分钟循环
   - 风格：宏大的管弦乐开场，逐渐加入电子元素
   - 情绪：欢迎、期待、冒险的召唤
   - 特点：可循环播放，不会产生疲劳感

2. **太空航行音乐**
   - 时长：5-8分钟循环
   - 风格：环境音乐，大量使用pad和氛围音效
   - 情绪：宁静、深邃、略带孤独感
   - 变化：根据航行距离和目的地调整音色

3. **贸易站音乐**
   - 时长：3-5分钟循环
   - 风格：轻快的电子音乐，加入人声采样
   - 情绪：繁忙、商业、机会
   - 特点：节奏明快，体现商业活动的活力

4. **探索音乐**
   - 时长：4-6分钟循环
   - 风格：悬疑、神秘，使用不和谐音程
   - 情绪：紧张、好奇、未知
   - 变化：根据发现内容动态调整

## 音效设计

### 音效分类

1. **界面音效**
   - 按钮点击：清脆、确认感
   - 菜单切换：流畅、方向感
   - 错误提示：明显但不刺耳
   - 成功确认：满足感、正面反馈
   - 通知提醒：引起注意但不打断

2. **游戏音效**
   - 飞船引擎：低频轰鸣，科技感
   - 超空间跳跃：能量聚集到释放的过程
   - 货物装卸：机械操作声
   - 交易完成：金币、成功的声音
   - 扫描探测：电子设备工作声

3. **环境音效**
   - 太空环境：深邃的低频背景
   - 星球大气：风声、气候相关
   - 贸易站：人声、机械声、活动声
   - 工业区：机器运转、生产声
   - 自然区：根据星球类型的自然声

4. **反馈音效**
   - 经验获得：上升、成长的音效
   - 等级提升：庆祝、成就的音效
   - 任务完成：满足、完成的音效
   - 发现新事物：惊喜、发现的音效
   - 危险警告：紧急、警示的音效

### 音效设计原则

1. **功能性原则**
   - 每个音效都有明确的功能目的
   - 音效与视觉反馈相配合
   - 提供清晰的状态信息
   - 增强用户操作的确定性

2. **一致性原则**
   - 相同功能使用相似音效
   - 音效风格与游戏整体风格统一
   - 音量和音色保持平衡
   - 建立玩家的听觉习惯

3. **非干扰性原则**
   - 音效不应掩盖重要信息
   - 避免过于频繁或刺耳的音效
   - 提供音效开关和音量控制
   - 考虑长时间游戏的舒适度

4. **情感增强原则**
   - 音效应增强游戏的情感体验
   - 通过音效传达游戏世界的特色
   - 营造沉浸感和代入感
   - 强化玩家的成就感和满足感

2. **音频资源管理**
   - 音频文件格式选择（OGG、WAV）
   - 压缩质量和文件大小平衡

### 音频管理器设计（Web 版）

1. **AudioManager类**
```ts
type BusName = "master" | "music" | "sfx" | "voice";

export class AudioManager {
   private ctx: AudioContext;
   private buses: Record<BusName, GainNode>;
   private bufferCache = new Map<string, AudioBuffer>();
   private currentMusic: AudioBufferSourceNode | null = null;

   constructor() {
      this.ctx = new AudioContext();
      const master = this.ctx.createGain();
      const music = this.ctx.createGain();
      const sfx = this.ctx.createGain();
      const voice = this.ctx.createGain();

      music.connect(master);
      sfx.connect(master);
      voice.connect(master);
      master.connect(this.ctx.destination);

      this.buses = { master, music, sfx, voice };
   }

   async loadBuffer(name: string, url: string): Promise<void> {
      if (this.bufferCache.has(name)) return;
      const res = await fetch(url);
      const arr = await res.arrayBuffer();
      const buf = await this.ctx.decodeAudioData(arr);
      this.bufferCache.set(name, buf);
   }

   playMusic(name: string, fadeSeconds = 1): void {
      const buf = this.bufferCache.get(name);
      if (!buf) return;
      this.stopMusic(fadeSeconds);

      const src = this.ctx.createBufferSource();
      src.buffer = buf;
      src.loop = true;
      src.connect(this.buses.music);
      src.start();
      this.currentMusic = src;
   }

   stopMusic(fadeSeconds = 0): void {
      if (!this.currentMusic) return;
      try {
         this.currentMusic.stop(this.ctx.currentTime + Math.max(0, fadeSeconds));
      } finally {
         this.currentMusic = null;
      }
   }

   playSfx(name: string): void {
      const buf = this.bufferCache.get(name);
      if (!buf) return;
      const src = this.ctx.createBufferSource();
      src.buffer = buf;
      src.connect(this.buses.sfx);
      src.start();
   }

   setBusVolume(bus: BusName, volume01: number): void {
      this.buses[bus].gain.value = Math.max(0, Math.min(1, volume01));
   }
}
```

2. **动态音乐系统（实现提示）**
- 将“状态机/强度”映射为：多个循环音轨（layer）+ 不同 `GainNode` 的淡入淡出。
- 使用 `AudioContext.currentTime` 做精确调度，避免 UI 线程抖动造成卡顿。

1. **技术规范**
   - 采样率：44.1kHz

   - 格式：OGG Vorbis（音乐）/ WAV（音效）
   - 音量标准：-23 LUFS（音乐）/ -18 dBFS（音效）

2. **质量标准**
   - 无明显失真和噪音
   - 频率响应平衡

   - 循环音乐无缝衔接
