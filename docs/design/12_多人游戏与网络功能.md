# 星际贸易商 - 多人游戏与网络功能

## 系统概述

多人游戏与网络功能为《星际贸易商》提供了社交互动和协作竞争的体验，让玩家能够在共享的星际贸易世界中相互交流、合作和竞争。本设计文档详细规划了多人游戏模式、网络架构、社交功能和技术实现方案。

## 多人游戏模式

### 游戏模式分类

1. **合作模式**
   - 联合贸易：多名玩家组成贸易联盟
   - 资源共享：共享货物、信息和设施
   - 协作任务：需要多人配合完成的大型任务
   - 共同目标：建立共同的贸易帝国

2. **竞争模式**
   - 市场竞争：争夺有限的贸易机会
   - 排行榜：财富、声望、成就等排名
   - 拍卖竞价：稀有物品的竞拍
   - 区域控制：争夺贸易路线和市场份额

3. **混合模式**
   - 派系对抗：不同阵营间的竞争与合作
   - 动态联盟：临时的合作关系
   - 事件响应：共同应对危机事件
   - 经济战争：通过经济手段进行对抗

### 服务器架构

1. **服务器类型**
   - 官方服务器：稳定的长期游戏世界
   - 私人服务器：玩家自建的定制服务器
   - 临时房间：短期的多人游戏会话
   - 本地网络：局域网内的多人游戏

2. **世界持久性**
   - 持续世界：24/7运行的游戏世界
   - 会话制：固定时间的游戏回合
   - 混合模式：部分持续、部分会话
   - 存档同步：跨服务器的进度同步

3. **玩家容量**
   - 小型服务器：2-8名玩家
   - 中型服务器：10-50名玩家
   - 大型服务器：100-500名玩家
   - 巨型服务器：1000+名玩家

## 网络架构设计

### 客户端-服务器架构

1. **服务器组件**
   - 游戏逻辑服务器：处理游戏规则和状态
   - 数据库服务器：存储玩家数据和游戏状态
   - 认证服务器：处理用户登录和权限
   - 匹配服务器：玩家匹配和房间管理

2. **客户端组件**
   - 游戏客户端：主要游戏界面和逻辑
   - 网络模块：处理网络通信
   - 缓存系统：本地数据缓存
   - 同步机制：状态同步和冲突解决

3. **通信协议**
   - TCP连接：可靠的数据传输
   - UDP包：实时性要求高的数据
   - WebSocket：Web平台的实时通信
   - HTTP API：非实时的数据交换

### 数据同步策略

1. **状态同步**
   - 权威服务器：服务器为最终权威
   - 客户端预测：减少延迟的本地预测
   - 状态回滚：处理预测错误
   - 增量更新：只传输变化的数据

2. **冲突解决**
   - 时间戳优先：基于时间的冲突解决
   - 服务器仲裁：服务器决定最终结果
   - 补偿机制：错误操作的补偿
   - 回滚重放：状态回滚和重新执行

3. **网络优化**
   - 数据压缩：减少网络传输量
   - 批量传输：合并多个小包
   - 优先级队列：重要数据优先传输
   - 自适应质量：根据网络状况调整

## 技术实现

### Web 网络能力

1. **通信方式**
   - WebSocket：客户端与服务器长连接（适合实时状态同步）
   - HTTP(S)：登录/配置/排行榜等请求-响应接口
   - WebRTC：P2P（可选；通常需要信令服务）

2. **传输与协议**
   - WebSocket（JSON 或二进制）：消息推送与同步
   - 自定义协议：建议先 JSON（便于调试），后期热点路径再用二进制

3. **状态管理**
   - 权威服务器（推荐）：服务器做最终裁决，客户端预测/插值
   - 状态同步：快照（snapshot）/增量（delta）/事件（event）三种策略组合
   - 反作弊：核心逻辑放在服务器校验

3. **API设计（Web 客户端示例）**
```ts
export type NetworkEvent =
   | { type: "player_connected"; id: number }
   | { type: "player_disconnected"; id: number }
   | { type: "message"; from: number; message: string };

export class NetworkClient {
   private ws: WebSocket | null = null;
   private listeners = new Set<(ev: NetworkEvent) => void>();

   connect(url: string): void {
      const ws = new WebSocket(url);
      ws.onmessage = (e) => {
         const msg = JSON.parse(String(e.data)) as NetworkEvent;
         for (const cb of this.listeners) cb(msg);
      };
      this.ws = ws;
   }

   onEvent(cb: (ev: NetworkEvent) => void): () => void {
      this.listeners.add(cb);
      return () => this.listeners.delete(cb);
   }

   send(payload: unknown): void {
      this.ws?.send(JSON.stringify(payload));
   }

   joinGuild(guildId: number): void {
      this.send({ type: "join_guild", guildId });
   }
}
```
