# 游戏设计文档

## 1. 游戏概述
### 1.1 游戏简介
《星际贸易商》是一款2D经营类游戏，玩家扮演一名星际贸易商，在浩瀚的银河系中穿梭，通过买卖各种商品积累财富，最终建立自己的商业帝国。

### 1.2 游戏背景
#### 1.2.1 世界观设定
游戏设定在未来的银河系，人类已经掌握了星际航行技术，并在众多星球上建立了殖民地。各个星球因为资源禀赋、技术水平和文化差异，形成了独特的经济体系和贸易需求。

#### 1.2.2 历史背景
在银河历3045年，随着曲速引擎技术的突破，星际旅行成本大幅降低，星际贸易进入了黄金时代。各大商业公司和独立贸易商纷纷崛起，在星际贸易中寻找机遇。

### 1.3 游戏目标
1. **财富积累**：通过贸易活动积累大量财富
2. **商业扩张**：拥有多艘高级飞船和多个贸易站
3. **声望提升**：在各大星球和贸易组织中获得高声望
4. **影响力扩大**：能够影响星球经济和政治走向

### 1.4 游戏特色
1. **自由开放的贸易世界**：玩家可以自由选择贸易路线和商品
2. **动态经济系统**：市场价格会根据供需关系实时变化
3. **多样化的发展路径**：可以选择成为和平贸易商、走私者、赏金猎人等不同路线
4. **丰富的随机事件**：遭遇海盗、发现稀有资源、遇到贸易机会等
5. **深度的管理系统**：管理飞船、船员、资源和贸易站

### 1.5 目标平台
- PC (Windows/Mac/Linux)
- Web（浏览器）实现：TypeScript + Canvas 2D（或 WebGL）

### 1.6 核心游戏循环

《星际贸易商》采用三层嵌套循环设计，确保玩家在不同时间尺度上都能获得明确的目标和成就感：

**微循环（5-10分钟）：** 单次贸易循环
- 发现需求 → 购买商品 → 航行 → 卖出 → 获利
- 即时反馈：利润显示、经验获取、技能提升

**中循环（30-60分钟）：** 贸易路线经营
- 规划路线 → 多次交易 → 升级飞船 → 解锁新星球
- 成长反馈：等级提升、声望变化、新区域解锁

**宏循环（数小时~数十小时）：** 商业帝国建设
- 建立贸易网 → 影响经济 → 派系抉择 → 剧情推进
- 长期目标：称号成就、世界观变化、结局分支

## 2. 核心玩法系统

### 2.1 贸易系统
#### 2.1.1 商品系统
##### 商品分类
1. **基础资源**
   - 矿物（铁矿、铜矿、稀有金属等）
   - 能源（燃料、反物质、能量晶体等）
   - 农产品（食物、药材、有机材料等）

2. **工业产品**
   - 零部件（引擎组件、电子元件、武器部件等）
   - 消费品（服装、家居用品、娱乐设备等）
   - 高科技产品（AI芯片、量子计算机、生物增强设备等）

3. **奢侈品**
   - 艺术品（绘画、雕塑、古董等）
   - 珍稀物品（外星生物标本、稀有宝石、限量收藏品等）
   - 高端服务（VIP通行证、星际旅游套餐等）

4. **特殊商品**
   - 走私品（违禁药物、未授权技术、武器等）
   - 情报数据（星图、技术蓝图、政治秘密等）
   - 任务物品（特定任务需要运输的物品）

##### 商品属性
- **基础价值**：商品的参考价格
- **体积**：占用货舱空间的大小
- **重量**：影响飞船速度和燃料消耗
- **稀有度**：决定商品在市场上的出现频率
- **保质期**：某些商品（如食物、药品）会随时间贬值
- **合法性**：商品在不同星球的法律地位
- **需求波动**：价格波动的幅度和频率

#### 2.1.2 市场系统
##### 市场类型
1. **公开市场**：所有玩家都可以访问的正规交易场所
2. **黑市**：交易违禁品，价格波动大，风险高
3. **专业市场**：只交易特定类别商品的市场
4. **拍卖行**：稀有物品的竞价交易

##### 价格机制
价格由以下因素动态决定：
1. **基础供需关系**：每个星球对不同商品有固定的生产和消费倾向
2. **市场库存**：当前市场上的商品数量影响价格
3. **玩家行为**：大量买入或卖出会影响市场价格
4. **随机事件**：战争、自然灾害、技术突破等事件影响价格
5. **季节性变化**：某些商品会有周期性价格波动
6. **星球特性**：星球的政治、经济状况影响价格水平

##### 市场信息
玩家可以获取的市场信息：
1. **基本价格**：当前星球的商品买卖价格
2. **历史趋势**：过去一段时间的价格变化图表
3. **市场新闻**：影响价格的重大事件通知
4. **商人情报**：付费获取的其他星球价格信息
5. **市场预测**：通过技能或设备预测价格走势

#### 2.1.3 贸易路线
##### 路线规划
玩家可以规划固定的贸易路线，包括：
1. **路线点**：经过的星球顺序
2. **交易计划**：在每个星球买入和卖出的商品
3. **风险评估**：路线上可能遇到的危险
4. **时间估算**：完成整个路线所需的时间
5. **利润预测**：基于当前价格的预期收益

##### 自动贸易
达到一定等级后，玩家可以设置自动贸易：
1. **雇佣副手**：执行预设的贸易路线
2. **设置交易条件**：如"当价格低于X时买入，高于Y时卖出"
3. **风险管理**：设置遇到危险时的应对策略

#### 2.1.4 贸易任务
##### 任务类型
1. **运输合同**：在指定时间内将货物从A点运到B点
2. **紧急配送**：高风险高回报的限时运输
3. **特殊订单**：收集特定商品组合的订单
4. **市场操控**：通过大量买入或卖出影响市场价格

##### 任务奖励
1. **金钱**：直接的现金报酬
2. **声望**：提高与特定派系的关系
3. **特殊物品**：稀有商品、飞船升级等
4. **特权**：获取特殊交易权限或市场信息

#### 2.1.5 贸易技能
玩家可以通过经验提升以下贸易技能：
1. **谈判**：提高买入价和卖出价的差价
2. **市场分析**：获取更准确的价格信息和预测
3. **走私技巧**：降低走私被发现的风险
4. **仓储管理**：提高货舱利用效率
5. **路线优化**：降低燃料消耗和旅行时间

### 2.2 飞船系统
#### 2.2.1 飞船类型
##### 基础分类
1. **轻型货船**
   - 特点：速度快、机动性好、货舱小、防御弱
   - 适用：初期游戏、短途贸易、高价值小体积商品运输
   - 代表型号："星际信使"、"快速猎手"

2. **中型货船**
   - 特点：速度适中、货舱中等、防御一般
   - 适用：中期游戏、中等距离贸易
   - 代表型号："商业先锋"、"星际旅者"

3. **重型货船**
   - 特点：速度慢、货舱大、防御强
   - 适用：后期游戏、大宗商品运输
   - 代表型号："银河巨轮"、"资源巨兽"

4. **特种飞船**
   - 走私船：隐形能力强、速度快、特殊货舱
   - 战斗货船：武装加强、货舱适中
   - 探索船：燃料效率高、探测范围大
   - 豪华商船：乘客舱位多、声望加成高

##### 飞船属性
1. **性能属性**
   - 最大速度：决定星际航行时间
   - 加速性能：影响起飞和降落效率
   - 机动性：影响规避危险的能力
   - 燃料效率：每单位距离消耗的燃料量
   - 跃迁能力：是否能使用星际跃迁点

2. **实用属性**
   - 货舱容量：可装载的最大货物量
   - 燃料容量：最大燃料储存量
   - 船员容量：可容纳的最大船员数
   - 装备槽位：可安装的设备数量
   - 乘客舱位：可搭载的乘客数量

3. **防御属性**
   - 护盾强度：抵御伤害的能力
   - 护盾恢复：护盾自动恢复速度
   - 装甲值：护盾耗尽后的防御能力
   - 结构完整性：飞船的总生命值
   - 隐形能力：降低被检测到的几率

4. **攻击属性**（如适用）
   - 武器槽位：可安装的武器数量
   - 武器能量：支持武器系统的能量池
   - 目标系统：武器瞄准精度

#### 2.2.2 飞船获取
##### 购买系统
1. **飞船市场**
   - 不同星球提供不同类型和等级的飞船
   - 核心工业星球提供更多高级飞船选择
   - 二手市场提供价格较低但状态不佳的飞船

2. **价格因素**
   - 基础价格：飞船型号的标准价格
   - 星球加成：不同星球的价格变动
   - 声望折扣：基于玩家在该星球的声望
   - 市场波动：基于游戏内经济状况

##### 特殊获取途径
1. **任务奖励**：完成特定任务获得独特飞船
2. **打捞修复**：发现并修复废弃飞船
3. **拍卖会**：参与稀有飞船的竞价拍卖
4. **继承**：游戏特定剧情中获得
5. **海盗击败**：击败海盗后有机会获得其飞船

#### 2.2.3 飞船定制与升级
##### 核心系统升级
1. **引擎系统**
   - 标准引擎：平衡型能源消耗和速度
   - 高速引擎：提高最大速度，增加能源消耗
   - 经济引擎：降低燃料消耗，减少速度
   - 隐形引擎：降低热信号，减少被探测几率

2. **货舱系统**
   - 容量扩展：增加最大货物容量
   - 特种货舱：允许运输特殊货物（易爆、生物等）
   - 隐藏货舱：用于走私，不被常规扫描发现
   - 冷藏舱：延长易腐商品的保质期

3. **防御系统**
   - 基础护盾：提供基本防护
   - 反动能护盾：对物理攻击有更好防护
   - 反能量护盾：对能量武器有更好防护
   - 自适应护盾：根据攻击类型调整防御
   - 装甲强化：提高物理防御能力

4. **能源系统**
   - 标准能源核心：平衡型能源输出
   - 高容量电池：增加能源储存
   - 快速充能器：提高能源恢复速度
   - 效率优化器：降低系统能源消耗

##### 辅助设备
1. **导航设备**
   - 星图增强器：扩大可见星图范围
   - 跃迁计算机：减少跃迁准备时间
   - 危险探测器：提前警告危险区域
   - 资源扫描仪：发现隐藏资源点

2. **交易设备**
   - 市场分析器：提供更详细的市场信息
   - 价格预测器：预测商品价格趋势
   - 货物压缩器：增加有效货舱空间
   - 自动交易系统：设置自动买卖条件

3. **防御设备**
   - 紧急修复系统：自动修复受损部位
   - 反扫描干扰器：降低被扫描几率
   - 逃生舱：在飞船毁灭时保存部分货物
   - 防海盗系统：降低被海盗攻击的几率

4. **生活设备**
   - 高级生活舱：提高船员效率
   - 医疗舱：加速船员恢复
   - 训练设施：提高船员经验获取
   - 豪华客舱：增加乘客满意度和小费

##### 武器系统（可选）
1. **能量武器**
   - 激光炮：精准但伤害一般
   - 等离子炮：伤害高但能耗大
   - 离子炮：干扰敌方系统

2. **实体武器**
   - 轨道炮：高穿透力
   - 导弹发射器：高爆发伤害
   - 采矿激光：可用于资源开采

3. **防御武器**
   - 点防御系统：拦截导弹
   - 电子战系统：干扰敌方瞄准
   - 牵引光束：控制小型飞船

#### 2.2.4 飞船管理
##### 维护系统
1. **日常维护**
   - 定期维护：防止性能下降
   - 燃料补给：补充飞行所需燃料
   - 损伤修复：修复战斗或事故损伤

2. **状态影响**
   - 磨损度：影响飞船性能和故障率
   - 系统状态：各系统的工作效率
   - 清洁度：影响某些特殊任务的接取

3. **维修选项**
   - 标准维修：恢复基本功能
   - 高级维修：完全恢复性能
   - 紧急修理：临时修复关键系统
   - 全面翻新：重置磨损度

##### 船员管理
1. **船员角色**
   - 驾驶员：提高飞行速度和机动性
   - 工程师：提高修理效率和系统性能
   - 安保人员：提高防御和战斗能力
   - 商务官：提高交易利润和谈判能力
   - 医疗官：提高船员恢复速度
   - 科学家：提高探索和研究效率

2. **船员属性**
   - 技能等级：影响工作效率
   - 忠诚度：影响留任概率和工作态度
   - 特殊特质：提供独特加成
   - 薪资要求：维持船员所需成本

3. **船员发展**
   - 经验获取：通过执行任务获得经验
   - 技能培训：花费资源提升特定技能
   - 忠诚建立：通过奖金和待遇提高忠诚度
   - 特殊事件：随机事件影响船员发展

#### 2.2.5 飞船舰队
##### 舰队管理
1. **舰队组成**
   - 最大飞船数量：基于玩家等级和技能
   - 舰队角色：为每艘飞船分配特定任务
   - 编队设置：设置飞船间的协同策略

2. **自动化操作**
   - 贸易路线：设置自动贸易路线
   - 护航任务：保护其他飞船
   - 资源收集：自动收集资源
   - 巡逻任务：监控特定区域

3. **舰队协同**
   - 资源共享：飞船间传输资源
   - 防御协作：共同应对威胁
   - 信息共享：扩大探测范围

### 2.3 经济系统
#### 2.3.1 经济模型
##### 基础经济结构
1. **供需系统**
   - 每个星球有独特的生产和消费模式
   - 生产受星球类型、资源和发展水平影响
   - 消费受人口、文化和特殊需求影响
   - 供需平衡决定基础价格和价格稳定性

2. **价值链**
   - 原材料 → 基础产品 → 高级产品的生产链
   - 每一步加工增加商品价值
   - 完整价值链分布在多个星球间
   - 玩家可以专注于价值链的特定环节

3. **经济层级**
   - 本地经济：单一星球内的经济活动
   - 区域经济：星区内的经济网络
   - 全球经济：整个游戏世界的经济系统
   - 各层级经济相互影响但有一定独立性

##### 动态经济因素
1. **生产周期**
   - 不同商品有不同生产周期
   - 周期性产出导致价格周期性波动
   - 季节性因素影响某些商品产量
   - 技术进步可能改变生产效率

2. **消费模式**
   - 基础需求：稳定且必须满足
   - 弹性需求：随经济状况波动
   - 奢侈需求：富裕时期增加
   - 紧急需求：特殊事件触发的临时高需求

3. **经济事件**
   - 繁荣：产量增加，消费增加，价格上涨
   - 衰退：产量减少，消费减少，价格下跌
   - 通货膨胀：所有商品价格普遍上涨
   - 通货紧缩：所有商品价格普遍下跌

4. **技术变革**
   - 新技术出现改变特定商品的供需关系
   - 老技术淘汰导致相关商品需求下降
   - 技术突破创造新的商品类别
   - 研发投入影响技术进步速度

#### 2.3.2 市场系统
##### 市场类型
1. **公开市场**
   - 特点：透明度高，价格相对稳定
   - 交易品：合法商品，标准化产品
   - 规则：受官方监管，交易税收正常
   - 位置：主要星球的贸易中心

2. **黑市**
   - 特点：隐蔽性高，价格波动大
   - 交易品：违禁品，走私货，稀有物品
   - 规则：无官方监管，可能有黑帮控制
   - 位置：城市阴暗角落，废弃区域

3. **专业市场**
   - 特点：针对特定商品类别的专业交易场所
   - 交易品：特定类别商品（如矿物交易所）
   - 规则：行业自律，专业标准
   - 位置：与相关产业密切的星球

4. **拍卖市场**
   - 特点：竞价交易，高价值物品
   - 交易品：稀有物品，收藏品，限量产品
   - 规则：竞价机制，可能需要资格审核
   - 位置：高端商业区，豪华星球

5. **期货市场**
   - 特点：交易未来交付的商品
   - 交易品：大宗商品，稳定产出的商品
   - 规则：合约交易，履约保证
   - 位置：金融中心，发达星球

##### 价格机制
1. **基础价格计算**
   - 基准价值：商品的内在价值
   - 供需调整：基于当前供需关系的价格调整
   - 地区系数：不同星球的价格倍率
   - 质量变异：基于品质的价格浮动

2. **价格波动因素**
   - 短期波动：日常交易量变化
   - 中期波动：生产周期和季节性因素
   - 长期波动：技术变革和经济周期
   - 随机波动：突发事件和市场情绪

3. **价格信息获取**
   - 基础信息：当前星球的实时价格
   - 市场报告：付费获取其他星球价格
   - 情报网络：通过关系获取价格预测
   - 市场分析工具：识别价格趋势和机会

4. **价格操纵机制**
   - 大量交易：通过大量买卖影响价格
   - 垄断控制：控制特定商品的供应
   - 信息操控：散布虚假信息影响市场
   - 政策影响：通过任务影响官方政策

##### 交易机制
1. **基本交易流程**
   - 查看市场：浏览可用商品和价格
   - 议价过程：可能的价格谈判（基于技能）
   - 交易执行：确认交易并支付/收款
   - 交易记录：记录交易历史和统计

2. **交易限制**
   - 数量限制：市场可提供的最大数量
   - 频率限制：市场刷新周期
   - 资格限制：某些市场需要特定声望或许可
   - 法律限制：违禁品交易的法律风险

3. **交易费用**
   - 交易税：官方市场的固定税率
   - 中介费：使用中介服务的费用
   - 保险费：高风险交易的保障费用
   - 贿赂：规避限制的非法支出

4. **合约交易**
   - 运输合约：将货物从A点运到B点
   - 供应合约：长期提供特定商品
   - 期货合约：预付款锁定未来价格
   - 特殊合约：完成特定条件的定制任务

#### 2.3.3 金融系统
##### 货币系统
1. **主要货币**
   - 星际信用点：通用货币，所有正规市场接受
   - 区域货币：特定星区使用的本地货币
   - 特殊代币：特定组织或黑市使用的交换媒介

2. **货币属性**
   - 汇率：不同货币间的兑换比例
   - 稳定性：货币价值的波动程度
   - 流通性：货币使用的广泛程度
   - 特殊功能：某些货币的独特用途

3. **货币获取**
   - 交易利润：买卖商品的差价
   - 任务奖励：完成各类任务
   - 服务收入：提供运输、保护等服务
   - 投资回报：各类投资的收益

##### 银行系统
1. **基础服务**
   - 存款：安全存储资金，可能有利息
   - 取款：在不同星球访问资金
   - 转账：向其他账户发送资金
   - 货币兑换：不同货币间的转换

2. **贷款系统**
   - 商业贷款：用于购买飞船或设备
   - 紧急贷款：临时资金周转
   - 投资贷款：用于大型项目融资
   - 信用等级：影响贷款额度和利率

3. **保险系统**
   - 货物保险：赔偿丢失或损坏的货物
   - 飞船保险：赔偿飞船损失
   - 生命保险：船员伤亡赔偿
   - 特殊险种：针对特定风险的保障

##### 投资系统
1. **直接投资**
   - 产业投资：投资星球上的生产设施
   - 贸易站投资：建立或升级贸易站
   - 研发投资：支持新技术研发
   - 基础设施：改善星球交通和设施

2. **金融投资**
   - 股票：购买公司股份获取分红
   - 债券：固定收益的借贷凭证
   - 期货：预测商品价格变动获利
   - 风险投资：高风险高回报的新项目

3. **回报机制**
   - 定期收益：固定周期获得收入
   - 价值增长：投资物本身价值增加
   - 特权获取：获得特殊访问权或折扣
   - 影响力：获得对经济决策的影响力

### 2.4 星球与探索系统
#### 2.4.1 星球类型
1. **核心工业星球**
   - 特点：高度发达的工业中心，人口密集
   - 经济：制造业发达，高科技产品产量高
   - 贸易机会：高级设备和技术产品的主要来源
   - 代表星球："新地球"、"铁锤星"

2. **农业星球**
   - 特点：广阔的耕地，适宜的气候
   - 经济：农产品和生物资源丰富
   - 贸易机会：食品、药材和有机材料的主要来源
   - 代表星球："绿洲IV"、"丰收星"

3. **采矿星球**
   - 特点：富含矿物资源，环境恶劣
   - 经济：矿业发达，原材料产量高
   - 贸易机会：各类矿物和原材料的主要来源
   - 代表星球："钻石岩"、"铁核星"

4. **研究星球**
   - 特点：先进的研究设施，学术氛围浓厚
   - 经济：研发产业发达，教育水平高
   - 贸易机会：科研数据、实验设备和高级技术
   - 代表星球："智慧枢纽"、"量子星"

5. **娱乐星球**
   - 特点：豪华的休闲设施，旅游业发达
   - 经济：服务业为主，奢侈品消费高
   - 贸易机会：艺术品、奢侈品和高端服务
   - 代表星球："极乐世界"、"星光大道"

6. **边缘殖民地**
   - 特点：新建立的殖民地，基础设施有限
   - 经济：发展初期，需求多样但规模小
   - 贸易机会：基础生活物资和建设材料
   - 代表星球："新希望"、"拓荒者站"

7. **废弃星球**
   - 特点：曾经繁荣现已废弃，危险但资源丰富
   - 经济：无正规经济，走私和探险者天堂
   - 贸易机会：稀有物品、古董和黑市商品
   - 代表星球："遗忘之城"、"幽灵星"

#### 2.4.2 星球属性
##### 基础属性
1. **物理特征**
   - 大小：影响资源丰富度和人口容量
   - 气候：影响适宜居住度和农业产出
   - 重力：影响起降难度和燃料消耗
   - 大气：决定是否需要特殊设备才能登陆

2. **经济属性**
   - 发展水平：影响可用设施和服务
   - 主导产业：决定主要产出和需求
   - 市场规模：影响交易量和价格稳定性
   - 税率：影响交易成本

3. **政治属性**
   - 政府类型：影响法律限制和贸易政策
   - 稳定性：影响市场波动和随机事件
   - 派系关系：影响与其他星球的贸易关系
   - 安全等级：影响犯罪率和海盗活动

4. **特殊属性**
   - 独特资源：只在该星球出现的特殊商品
   - 文化特色：影响特殊任务和交互选项
   - 科技专长：特定领域的技术优势
   - 隐藏区域：需要探索才能发现的秘密地点

#### 2.4.3 星系结构
##### 星区划分
1. **核心星区**
   - 特点：高度发达，安全系数高
   - 星球密度：高，星球间距离短
   - 贸易特点：竞争激烈，利润空间小但稳定
   - 政治：中央政府控制严格

2. **中层星区**
   - 特点：发展中，安全系数中等
   - 星球密度：中等，有一定探索空间
   - 贸易特点：平衡的风险和回报
   - 政治：地方政府自治，中央影响有限

3. **边缘星区**
   - 特点：开发初期，安全系数低
   - 星球密度：低，大片未探索区域
   - 贸易特点：高风险高回报
   - 政治：无政府状态或小型独立政权

4. **禁区**
   - 特点：极度危险，未被官方认可的区域
   - 星球密度：未知，多为特殊天体
   - 贸易特点：黑市和非法交易
   - 政治：海盗、犯罪组织或神秘势力控制

##### 航行路线
1. **主干航线**
   - 连接主要星球的官方航线
   - 安全有保障，燃料站和维修设施完善
   - 交通繁忙，海盗活动少

2. **次级航线**
   - 连接次要星球的常规航线
   - 基础设施一般，安全系数中等
   - 交通适中，偶有海盗活动

3. **危险航线**
   - 通往边缘区域或特殊区域的航线
   - 基础设施匮乏，安全系数低
   - 交通稀少，海盗活动频繁

4. **秘密航线**
   - 需要特殊情报才能发现的隐藏航线
   - 可能提供捷径或通往特殊区域
   - 状况不明，风险和机遇并存

#### 2.4.4 探索系统
##### 探索机制
1. **星图绘制**
   - 初始星图只显示已知区域
   - 通过探索扩展可见星图范围
   - 可购买部分区域的星图数据
   - 探索完成度影响游戏成就和特殊奖励

2. **扫描系统**
   - 基础扫描：发现星球和基本资源
   - 详细扫描：揭示星球详细信息和特殊资源
   - 深度扫描：发现隐藏区域和稀有资源

3. **探索点**
   - 异常信号：可能是资源、遗迹或危险
   - 太空遗迹：古代文明的遗留物
   - 自然奇观：独特的宇宙现象
   - 废弃站点：被遗弃的空间站或殖民地

4. **探索风险**
   - 太空异常：辐射区、引力异常、时空扭曲等
   - 自然灾害：小行星带、太阳风暴、黑洞等
   - 未知生物：外星生物体或寄生虫
   - 防御系统：古代遗迹的自动防御装置

##### 探索奖励
1. **资源发现**
   - 稀有矿藏：高价值矿物
   - 生物样本：独特的外星生物
   - 能源源泉：特殊能源资源

2. **知识获取**
   - 科技蓝图：解锁新技术或设备
   - 历史数据：揭示宇宙背景故事
   - 星图碎片：扩展可见星图

3. **物品获取**
   - 古代文物：高价值收藏品
   - 先进设备：独特功能的装备
   - 神秘物品：具有特殊效果的物品

4. **地点发现**
   - 隐藏基地：可作为个人据点
   - 秘密市场：交易特殊商品的场所
   - 安全避风港：危险区域中的安全地点

#### 2.4.5 星球互动
##### 着陆系统
1. **着陆条件**
   - 大气条件：影响着陆难度
   - 重力影响：影响燃料消耗
   - 天气系统：特殊天气可能阻止着陆
   - 许可要求：某些星球需要特殊许可

2. **着陆点类型**
   - 太空港：完善的设施和服务
   - 简易停机坪：基础设施有限
   - 野外着陆：无设施，可能有风险
   - 秘密基地：隐藏的着陆点，特殊功能

3. **着陆费用**
   - 停泊费：基于飞船大小和停留时间
   - 服务费：使用维修、补给等服务
   - 入境税：某些星球征收的特殊税费

##### 星球设施
1. **贸易设施**
   - 商品交易所：主要交易场所
   - 黑市：交易非法或稀有商品
   - 拍卖行：竞价获取稀有物品
   - 仓储设施：存储货物的场所

2. **服务设施**
   - 维修厂：修复飞船损伤
   - 改装车间：升级飞船设备
   - 燃料站：补充飞船燃料
   - 医疗中心：治疗船员伤病

3. **信息设施**
   - 情报中心：获取任务和信息
   - 导航站：更新星图数据
   - 研究实验室：分析探索发现
   - 通讯中心：与其他星球联系

4. **社交设施**
   - 酒吧：获取非正式情报和任务
   - 休闲中心：提高船员士气
   - 招聘中心：雇佣新船员
   - 派系总部：提升特定派系声望

#### 2.4.6 派系系统
##### 主要派系
1. **星际联邦**
   - 理念：秩序、法律、集体利益
   - 控制区域：核心星区
   - 特色：先进技术，严格监管
   - 贸易特点：稳定但利润有限

2. **自由贸易联盟**
   - 理念：自由市场，最小干预
   - 控制区域：主要贸易星球和航线
   - 特色：商业创新，低税收
   - 贸易特点：竞争激烈，机会多样

3. **边缘殖民者联合**
   - 理念：独立自主，互助合作
   - 控制区域：边缘星区殖民地
   - 特色：资源丰富，基础设施有限
   - 贸易特点：高需求，高风险高回报

4. **科技研究协会**
   - 理念：科学进步，知识共享
   - 控制区域：研究星球和科技设施
   - 特色：先进技术，严格保密
   - 贸易特点：高科技产品，研究数据

### 2.5 任务与剧情系统
#### 2.5.1 任务类型
1. **主线任务**
   - 推动核心剧情发展的关键任务
   - 解锁游戏重要功能和区域
   - 展现玩家角色的成长历程
   - 通常有较高难度和丰厚奖励

2. **派系任务**
   - 与特定派系相关的任务链
   - 提高与该派系的关系
   - 揭示派系背景和内部运作
   - 解锁派系特殊权限和装备

3. **贸易任务**
   - 运输特定货物到指定地点
   - 在限定时间内完成交付
   - 处理特殊货物的挑战
   - 建立贸易网络和声誉

4. **探索任务**
   - 发现新星球和航线
   - 调查异常现象和信号
   - 绘制未知区域的星图
   - 寻找失落的宝藏和遗迹

5. **战斗任务**（可选）
   - 护送重要货物或人物
   - 清除航线上的海盗威胁
   - 参与派系冲突
   - 防御攻击和突袭

6. **调查任务**
   - 收集情报和信息
   - 追踪特定人物或物品
   - 解开谜题和密码
   - 揭露阴谋和秘密

7. **救援任务**
   - 救助遇险飞船
   - 运送医疗物资和救援设备
   - 疏散危险区域的居民
   - 寻找失踪人员

8. **特殊事件任务**
   - 响应随机触发的紧急情况
   - 参与限时的特殊活动
   - 应对自然灾害或危机
   - 抓住稀有的商业机会

#### 2.5.2 任务结构
##### 任务组成
1. **基本信息**
   - 任务名称和描述
   - 目标和完成条件
   - 时间限制（如适用）
   - 难度等级和推荐装备
   - 奖励和后果

2. **任务链**
   - 相互关联的一系列任务
   - 前置任务和后续任务
   - 分支选择和多结局
   - 长期发展的故事线

3. **任务状态**
   - 可接取：满足前置条件
   - 进行中：已接取未完成
   - 已完成：满足完成条件
   - 失败：未能满足条件或超时
   - 锁定：未满足前置条件

4. **任务追踪**
   - 当前目标提示
   - 目标位置导航
   - 进度跟踪和计时
   - 相关信息和提示

##### 任务获取
1. **任务来源**
   - 任务板：公开发布的任务
   - NPC对话：通过交谈获得
   - 通讯请求：远程联系提供
   - 探索发现：调查异常信号
   - 声望解锁：达到特定声望等级

2. **任务筛选**
   - 按类型筛选：贸易、探索等
   - 按难度筛选：初级到高级
   - 按派系筛选：特定派系任务
   - 按奖励筛选：信用点、物品等

3. **任务刷新**
   - 定时刷新：按游戏时间周期
   - 位置刷新：不同星球不同任务
   - 事件刷新：特定事件触发
   - 声望刷新：声望变化带来新任务

##### 任务奖励
1. **基础奖励**
   - 信用点：游戏通用货币
   - 经验值：提升玩家等级
   - 声望点：提高与特定派系关系
   - 物品：装备、资源或特殊物品

2. **特殊奖励**

## 5. 美术风格

《星际贸易商》采用像素艺术风格，融合复古科幻美学和现代UI设计。

**像素分辨率标准：**
- 飞船精灵：32×32或64×64像素
- UI图标：16×16像素
- 星球缩略图：128×128像素
- 背景星空：平铺纹理+视差滚动

**色彩规范：**
- 太空背景：深蓝渐变（#0A0E27 ~ #1A1E47）
- 星球类型色系：
  - 工业星：灰蓝色 #6B7B8D
  - 农业星：绿色 #4CAF50
  - 采矿星：棕橙色 #8D6E48
  - 研究星：紫色 #7B68AE
  - 娱乐星：金色 #FFD700
  - 边缘殖民地：黄绿色 #8BC34A
  - 废弃星：暗红色 #5C3A3A
- UI主色调：深蓝（太空）、青色（科技）、橙色（商业）
- 功能色：绿色（确认）、红色（警告）、黄色（注意）、蓝色（信息）

**动画规格：**
- 飞船引擎尾焰：3-5帧循环，12 FPS
- UI面板展开：缓动动画，0.2秒
- 星球自转：慢速循环精灵表，4-8帧
- 交易成功：粒子+闪光效果

**参考风格：**
- 《星际牛仔》的复古科幻感
- 《FTL》的简洁2D太空风格
- 像素画家Waneella的科幻场景氛围

### 2.6 用户界面与体验设计
#### 2.6.1 设计理念
##### 核心原则
1. **直观性**
   - 界面元素的功能应一目了然
   - 重要信息和控制应突出显示
   - 使用通用的设计模式和符号
   - 减少学习曲线和认知负担

2. **一致性**
   - 在整个游戏中保持一致的设计语言
   - 类似功能使用类似的交互方式
   - 颜色、图标和布局的统一标准
   - 预测性行为和反馈

3. **效率性**
   - 减少完成常见任务所需的步骤
   - 提供快捷键和高级控制选项
   - 关键信息一目了然
   - 减少不必要的确认和过渡

4. **沉浸感**
   - 界面设计符合游戏世界观
   - 视觉和音效增强用户体验
   - 减少破坏游戏流程的干扰
   - 情境化的反馈和提示

##### 视觉风格
1. **科技感**
   - 未来主义的界面元素
   - 全息投影效果
   - 数字化显示和动画
   - 精密和高科技的细节

2. **色彩方案**
   - 主色调：深蓝色（代表太空）
   - 强调色：青色（代表科技）和橙色（代表商业）
   - 功能色：绿色（正面/确认）、红色（负面/警告）
   - 中性色：灰色色阶（背景和次要信息）

3. **图标设计**
   - 简洁明了的线条图标
   - 功能一致的视觉语言
   - 易于识别的形状和符号
   - 适当的细节和区分度

4. **排版**
   - 清晰易读的无衬线字体
   - 层次分明的文本大小
   - 适当的行距和字距
   - 重要信息的强调处理

#### 2.6.2 界面布局
##### 主界面结构
1. **HUD（平视显示器）**
   - 位置：屏幕边缘，不干扰中央视图
   - 内容：关键状态信息（资金、位置、时间等）
   - 特点：半透明，可折叠，情境敏感

2. **导航栏**
   - 位置：屏幕顶部或底部
   - 内容：主要功能区域的快速访问
   - 特点：图标+文字，高亮当前区域

3. **工作区**
   - 位置：屏幕中央主要区域
   - 内容：当前功能的详细界面
   - 特点：可定制布局，多面板支持

4. **快捷工具栏**
   - 位置：屏幕边缘或角落
   - 内容：常用功能的快速访问
   - 特点：可自定义，带快捷键提示

1. **分辨率适应**

#### 音频管理器设计
1. **AudioManager类**
```ts
type AudioBus = "music" | "sfx" | "voice";

export class AudioManager {
   private ctx = new AudioContext();
   private master = this.ctx.createGain();
   private buses: Record<AudioBus, GainNode> = {
      music: this.ctx.createGain(),
      sfx: this.ctx.createGain(),
      voice: this.ctx.createGain(),
   };
   private bufferCache = new Map<string, AudioBuffer>();

   constructor() {
      this.master.connect(this.ctx.destination);
      this.buses.music.connect(this.master);
      this.buses.sfx.connect(this.master);
      this.buses.voice.connect(this.master);
   }

   async resumeIfNeeded(): Promise<void> {
      if (this.ctx.state !== "running") await this.ctx.resume();
   }

   async loadBuffer(url: string): Promise<AudioBuffer> {
      const cached = this.bufferCache.get(url);
      if (cached) return cached;

      const res = await fetch(url);
      const arr = await res.arrayBuffer();
      const buf = await this.ctx.decodeAudioData(arr);
      this.bufferCache.set(url, buf);
      return buf;
   }

   async playSfx(url: string, bus: AudioBus = "sfx"): Promise<void> {
      await this.resumeIfNeeded();
      const buffer = await this.loadBuffer(url);
      const src = this.ctx.createBufferSource();
      src.buffer = buffer;
      src.connect(this.buses[bus]);
      src.start();
   }

   setBusVolume(bus: AudioBus, volume01: number): void {
```
